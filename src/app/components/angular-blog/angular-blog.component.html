    Creating an Approuting module

    Sometimes when a new angular project is created an app-routing module may not be generated which is essential to handle navigation from one component page to another.

  • run the command ng g module app-routing --flat --module=app

    • Setting Angular Routing Paths

    Routing paths setting and usage basically involves these steps below

  • In the app-routing.module.ts file import all the components you will like to route to for example: import HomeComponent from './components/home/home.component'; remember to include the curly braces around HomeComponent
  • import Routes,Router,RouterModule from'@angular/router';
  • In the exports section of your app-routing.module.ts file inject the RouterModule.
  • create an array constant routes and set it to a type of Routes, for example const routes:Routes=[]
  • specify the route path and component while enclosing it in a curly braces e.g path:'home', component:HomeComponent
  • To navigate tothe home component from any component in your angular project simply set the routerLink path to home as in /home

    • Handling Search Exceptions In Angular Projects

    Users may erroneously search for a page that does not exist in your project, and forget how to navigate back, you can handle these exceptions using a notfoundcomponent.

    Follow these steps below in order to create a notfound component, inform the user of his/her possible error and kindly suggest a redirection to your home page.

  • ng g c components/notfound
  • in the notfound.component.html file create an easy to understand directive such as "You Might Want to Rephrase Your Search or check here " set the routerLink path to your home page.
  • In the path creation for the notfound component set its value to '**' these simply handles all exceptions.
